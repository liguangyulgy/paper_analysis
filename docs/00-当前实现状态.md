# 00-当前实现状态

## 1. 说明

本项目目前处于早期 MVP 建设阶段。`docs/` 中的大部分文档是产品和工程设计，不代表相关命令或功能已经全部实现。

当前实现重点是搭建可维护的本地文献数据库地基，并收敛 MVP-1 的最小闭环。

## 2. 已实现能力

当前代码已实现：

- Python 工程骨架；
- SQLite schema 初始化；
- repository 层基础读写；
- DOI / PMID / PMCID / title 去重入口；
- 关键词配置读取；
- title / abstract 关键词命中记录；
- 状态汇总 CLI；
- 关键词命中统计 CLI；
- PubMed 按关键词获取摘要并入库；
- 简单 CSV / Markdown 论文清单导出；
- 状态变更事件表；
- 基础单元测试。

当前可运行命令：

```bash
python -m src.cli init db
python -m src.cli status summary
python -m src.cli status keywords
python -m src.cli collect abstracts --source pubmed --query "genOway" --limit 20
python -m src.cli report simple --format csv
python -m src.cli report simple --format markdown
python -m pytest
```

## 4. 当前 MVP-1 开发目标

MVP-1 只聚焦最小可运行文献题录闭环：

1. PubMed 按关键词获取摘要；`已实现`
2. 论文题录入库；`已实现`
3. DOI / PMID / PMCID / title 去重；`已实现`
4. 关键词命中记录；`已实现`
5. 状态汇总；`已实现`
6. 最简单 CSV 或 Markdown 导出。`已实现`

MVP-1 不要求自动全文下载，不要求 PDF 解析，不要求 WebUI。
