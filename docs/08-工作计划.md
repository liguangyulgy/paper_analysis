# 08-工作计划

## 1. 第一阶段：基础工程与摘要库

目标：先建立可运行的论文信息库。

任务：

1. 创建 Python 工程结构；
2. 建立 SQLite 数据库；
3. 实现 `papers`、`paper_sources`、`paper_fulltexts`、`paper_evidence`、`paper_external_links`、`manual_notes` 表；
4. 实现 PubMed 摘要获取；
5. 实现 Crossref DOI 补全；
6. 实现去重逻辑；
7. 实现基础 CLI；
8. 实现日志、限速、重试。

验收标准：

- 可以通过 CLI 按关键词获取论文；
- 可以保存题名、摘要、DOI、PMID、期刊、年份；
- 可以重复运行且不会重复入库；
- 可以查看论文状态汇总。

## 2. 第二阶段：开放全文获取与外部链接

目标：自动寻找合法开放全文，并生成外部访问入口。

任务：

1. 实现 Unpaywall 查询；
2. 根据 DOI 查找开放 PDF；
3. 根据 PMCID 查找 XML 全文；
4. 实现开放 PDF 下载；
5. 保存下载来源、license、oa_status、hash；
6. 更新 fulltext_status；
7. 对无法获取全文的论文记录原因；
8. 实现 `link_builder.py`；
9. 生成 DOI、PubMed、PMC、Unpaywall、Semantic Scholar、OpenAlex、Google Scholar 检索等外部链接。

验收标准：

- 可以对指定 paper_id 查找全文；
- 可以批量处理已获取摘要的论文；
- 不下载非开放全文；
- 全文状态准确可查询；
- 每篇论文能生成外部链接候选。

## 3. 第三阶段：手动全文导入与解析

目标：支持用户手动补充 PDF 并分析。

任务：

1. 实现手动 PDF 导入；
2. 支持按 paper_id、DOI、标题匹配已有论文；
3. 无法匹配时创建 manual_only 记录；
4. 实现 PDF 文本解析；
5. 保存解析文本；
6. 解析失败时记录状态；
7. CAJ 文件可上传登记，但标记为 conversion_needed。

验收标准：

- 用户可以导入单个 PDF；
- 用户可以批量导入目录；
- 手动 PDF 能与已有论文匹配；
- 解析结果可以进入后续分析流程；
- CAJ 文件不会误解析，会进入待转换清单。

## 4. 第四阶段：证据抽取与价值评分

目标：从摘要和全文中提取有业务价值的信息。

任务：

1. 实现关键词和规则抽取；
2. 抽取 genOway / 基锘威证据句；
3. 抽取模型名称、模型类型、靶点、疾病领域；
4. 抽取应用场景；
5. 抽取机构和地区；
6. 识别中国境内应用线索；
7. 计算 evidence_level；
8. 计算 reference_value；
9. 对不确定结果标记 needs_review；
10. 支持 manual_override 保护人工确认结果。

验收标准：

- 能识别高相关论文；
- 能区分强证据和弱证据；
- 能生成待人工审核列表；
- 人工可以修订 evidence_level 和 reference_value；
- 所有高价值候选都有证据句。

## 5. 第五阶段：报告导出

目标：形成可直接使用的分析报告。

任务：

1. 实现 Excel 报告；
2. 实现 Markdown 报告；
3. 输出新增论文清单；
4. 输出高参考价值论文清单；
5. 输出模型应用分析；
6. 输出疾病领域分析；
7. 输出待人工审核清单；
8. 输出无法获取全文清单；
9. 输出中国境内应用线索；
10. 输出 CAJ 待转换清单。

验收标准：

- 可以一键导出周报；
- 可以一键导出高价值论文报告；
- 报告中包含证据句和状态；
- 报告能支持市场、销售、产品规划使用。

## 6. 第六阶段：Streamlit WebUI 审核工作台

目标：提高人工审核和状态维护效率。

任务：

1. 实现 Dashboard 首页；
2. 实现论文列表和筛选；
3. 实现论文详情页；
4. 实现状态修改和人工备注；
5. 实现外部链接按钮；
6. 实现 PDF 上传；
7. 实现报告下载；
8. 实现中文数据源检索入口。

验收标准：

- 可以通过 WebUI 查看待审核论文；
- 可以查看摘要、证据句和全文状态；
- 可以点击 Google Scholar、DOI、PubMed、Unpaywall 等外部入口；
- 可以上传 PDF 并关联论文；
- 可以修改 reference_value、evidence_level、review_status；
- 人工修改会设置 manual_override；
- 可以下载报告。

## 7. 第七阶段：中文数据源支持

目标：支持中国境内应用情况研究。

任务：

1. 增加 `configs/chinese_queries.yaml`；
2. WebUI 增加 CNKI、万方、维普、SinoMed、NSTL 检索入口；
3. 支持中文论文手动录入；
4. 支持中文题录 CSV 导入；
5. 支持中文 PDF 上传；
6. 支持 CAJ 上传登记；
7. 报告中增加中国境内应用线索章节。

验收标准：

- 可以手动录入中文论文；
- 可以导入中文题录；
- 可以上传中文 PDF；
- CAJ 文件能进入待转换清单；
- 可以导出中国境内应用线索报告。
