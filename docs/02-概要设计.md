# 02-概要设计

## 1. 总体架构

系统采用本地 Python 工程实现，第一版使用 SQLite 存储数据，CLI 作为主要批处理入口。第二阶段增加 Streamlit WebUI，用于人工审核、状态维护、外部链接跳转、PDF 上传和报告查看。

总体流程：

```text
论文发现 / 摘要获取
        ↓
元数据标准化与去重
        ↓
全文发现与外部链接生成
        ↓
自动开放全文下载 / 用户手动下载并上传
        ↓
全文解析
        ↓
证据抽取与价值评分
        ↓
人工审核 / 状态修订
        ↓
报告导出
```

## 2. 模块划分

### 2.1 collectors：论文信息采集模块

负责从外部接口获取论文元数据和摘要。

第一阶段：

- `pubmed_collector.py`
- `crossref_collector.py`
- `unpaywall_collector.py`

第二阶段：

- `europe_pmc_collector.py`
- `openalex_collector.py`
- `semantic_scholar_collector.py`
- `scholar_alert_email_importer.py`

中文数据源第一版不做后台自动采集，只提供外部检索入口和手动导入。

### 2.2 fulltext：全文获取模块

负责根据 DOI / PMID / PMCID 寻找合法开放全文。

核心文件：

- `fulltext_resolver.py`
- `open_pdf_downloader.py`
- `pmc_xml_downloader.py`
- `manual_pdf_importer.py`
- `link_builder.py`

全文获取优先级：

```text
PMCID → XML
DOI → Unpaywall
Semantic Scholar openAccessPdf
OpenAlex open access location
出版社开放页面
手动导入
```

`link_builder.py` 负责为 WebUI 生成外部链接候选，包括 DOI、PubMed、PMC、Unpaywall、Semantic Scholar、Google Scholar 检索、出版社页面等。

### 2.3 parsers：文本解析模块

负责从摘要、XML、PDF、TXT、MD 中提取可分析文本。

核心文件：

- `abstract_parser.py`
- `xml_fulltext_parser.py`
- `pdf_parser.py`
- `section_extractor.py`
- `caj_handler.py`

`caj_handler.py` 第一版不解析 CAJ，只识别文件类型并标记 `conversion_needed`。

重点抽取章节：

- Abstract；
- Introduction；
- Materials and Methods；
- Methods；
- Animal Model；
- Mice；
- Rats；
- Results；
- Funding；
- Acknowledgements；
- Conflict of Interest；
- Figure Captions；
- Table Captions。

### 2.4 analysis：分析模块

负责抽取证据、识别模型应用、评分和状态维护。

核心文件：

- `evidence_extractor.py`
- `model_application_classifier.py`
- `china_relevance_classifier.py`
- `value_scorer.py`
- `paper_status_manager.py`

输出内容：

- evidence_level；
- reference_value；
- model_name；
- model_type；
- target；
- disease_area；
- application_scenario；
- institution；
- region_relevance；
- evidence_sentence；
- review_status。

### 2.5 webui：WebUI 审核工作台

第二阶段新增。

核心文件：

- `webui/streamlit_app.py`
- `webui/pages/dashboard.py`
- `webui/pages/paper_list.py`
- `webui/pages/paper_detail.py`
- `webui/pages/manual_upload.py`
- `webui/pages/reports.py`
- `webui/pages/chinese_sources.py`

WebUI 不替代 CLI。CLI 负责批量任务，WebUI 负责人工审核和操作。

### 2.6 reports：报告模块

负责输出 Excel、Markdown 和 CSV 报告。

核心文件：

- `excel_report.py`
- `markdown_report.py`
- `dashboard_data_exporter.py`

### 2.7 storage：存储模块

负责数据库连接、schema 初始化和数据访问。

核心文件：

- `database.py`
- `schema.sql`
- `repository.py`

## 4. 增量机制

系统必须支持增量运行。

摘要获取：

```text
按 source 维护 last_run_at，只拉取新增或更新记录。
```

全文获取：

```text
只处理 abstract_status = fetched 且 fulltext_status 为 not_checked / failed 的论文。
```

全文分析：

```text
只处理 parse_status = parsed 且 analysis_status 为 not_started / failed 的论文。
```

报告导出：

```text
默认只输出报告周期内新增或更新的论文。
```

人工覆盖保护：

```text
当 manual_override = true 时，自动任务不得覆盖 evidence_level、reference_value、review_status 等人工确认字段。
```
