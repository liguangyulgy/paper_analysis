# 03-状态机设计

## 1. 状态设计原则

不要只使用一个总状态字段。每篇论文需要维护多个维度的状态，便于增量处理、人工修订和报告筛选。

## 2. discovery_status

表示论文是否被发现、是否重复、是否排除。

```text
discovered
duplicate
excluded
```

## 3. abstract_status

表示摘要获取状态。

```text
not_checked
fetched
missing
failed
```

## 4. fulltext_status

表示全文获取状态。

```text
not_checked
resolving
xml_available
pdf_available
downloaded
unavailable
restricted
failed
conversion_needed
```

说明：

- `unavailable`：没有找到合法开放全文；
- `restricted`：疑似需要订阅或登录；
- `failed`：技术失败，如网络错误、解析异常；
- `conversion_needed`：CAJ 等暂不支持格式，需要用户转换为 PDF。

## 5. parse_status

表示全文解析状态。

```text
not_started
pending
parsed
partial
failed
conversion_needed
```

## 6. analysis_status

表示分析状态。

```text
not_started
pending
analyzed
failed
```

## 7. evidence_level

表示与 genOway / 基锘威的证据强度。

```text
unknown
A
B
C
D
X
```

含义：

```text
unknown = 尚未分析或证据不足，不能判断相关性
A = 正文或方法学明确说明模型来自 genOway / 基锘威
B = 全文明确提到 genOway / 基锘威，但模型关系需要确认
C = 只在作者单位、资助、利益冲突中出现
D = 只命中模型名或关键词，不能证明关联
X = 明确无关
```

默认值应为 `unknown`，避免把尚未分析的论文误判为弱相关。

## 8. reference_value

表示业务参考价值。

```text
high
medium
low
exclude
unknown
```

含义：

```text
high = 适合销售、市场、产品规划引用
medium = 有一定参考价值
low = 弱相关，仅保留线索
exclude = 明确无关或重复
unknown = 信息不足
```

## 9. review_status

表示人工审核状态。

```text
not_required
needs_review
reviewed
confirmed
rejected
```

## 10. language

表示论文语言。

```text
en
zh
mixed
unknown
```

## 11. region_relevance

表示是否与中国境内应用有关。

```text
china_domestic
global
unknown
not_china_related
```

`china_domestic` 的判断依据包括：

- 作者单位在中国境内；
- 中文数据源检索得到；
- 正文或摘要提到中国境内机构；
- 用户人工标记为中国境内线索。

## 12. manual_override

人工覆盖标志。

```text
true
false
```

当 `manual_override = true` 时，自动任务不得覆盖以下字段：

- evidence_level；
- reference_value；
- review_status；
- region_relevance；
- 用户手动备注；
- 人工确认的模型、疾病、应用场景字段。

## 13. 用户可见标签

系统可以根据状态自动生成标签：

```text
已发现
已获取摘要
摘要缺失
已发现开放全文
已获取全文
全文受限
全文解析失败
CAJ 待转换
已完成全文分析
参考价值高
参考价值中
参考价值低
需要人工审核
已人工确认
已排除
手动全文
Google Scholar 线索
中文文献线索
中国境内应用线索
```

## 14. PubMed genOway 证据字段

PubMed 记录中，`genOway` 很多时候出现在作者单位 `Affiliation`，而不是标题或摘要正文。

因此：

- PubMed 采集必须保存 authors 和 affiliations；
- 关键词命中应扫描 title、abstract 和 affiliations；
- 报告应输出 `genoway_evidence`；
- 不能因为摘要中没有 `genOway` 就判断论文没有 genOway 线索；
- 也不能因为 affiliations 中有 `genOway` 就判断论文使用了 genOway 小鼠。

证据类型应区分：

```text
supplier_usage = 明确来源/供应商证据，例如 from/provided by/generated by genOway
model_usage = 模型使用证据，但供应商关系需要确认
author_affiliation = genOway 只出现在作者单位或合作机构
keyword_only = 只命中模型名或泛关键词
```

其中 `author_affiliation` 默认只能对应 evidence_level = C，不能进入“明确使用 genOway 模型”清单。
