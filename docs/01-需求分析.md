# 01-需求分析

## 1. 背景

用户希望持续跟踪 genOway / 基锘威公司的动物模型应用情况，从公开论文、摘要、开放全文、中文数据源线索和用户手动整理的全文中提取有效信息，并形成可维护的数据资产和分析报告。

系统重点关注动物模型的实际应用，而不是简单收集论文列表。

## 2. 建设目标

系统需要支持四类核心能力：

1. 自动获取论文信息及摘要；
2. 自动获取开放论文全文并分析；
3. 分析用户手动整理的论文全文；
4. 整理并导出分析报告。

在第二阶段增加 WebUI 审核工作台，用于：

- 人工审核系统自动发现的论文；
- 查看论文摘要、证据句和全文状态；
- 点击外部链接跳转 DOI、PubMed、PMC、Unpaywall、Semantic Scholar、Google Scholar 等页面；
- 手动上传已下载 PDF；
- 修改论文状态、证据等级和参考价值；
- 导出报告。

在中国境内应用研究场景下，系统还需要支持中文数据源线索，包括 CNKI、万方、维普、SinoMed、NSTL 等。

## 3. 核心业务问题

系统最终应支持回答：

- 哪些论文与 genOway / 基锘威相关？
- 是否明确使用了相关动物模型？
- 具体模型名称是什么？
- 模型类型是什么？
- 涉及哪些靶点、疾病领域和治疗方向？
- 应用场景是药效、安全性、PK/PD、机制研究还是其他？
- 哪些论文参考价值高？
- 哪些论文需要人工审核？
- 哪些论文只有摘要，没有全文？
- 哪些论文全文受限，需要用户手动补充？
- 中国境内哪些高校、医院、科研机构、药企或 CRO 可能使用过相关模型？
- 中文论文、硕博论文、会议论文中是否存在有价值线索？

## 4. 数据来源

### 4.1 英文主数据源

第一版优先支持：

- PubMed；
- Crossref；
- Unpaywall；
- 用户手动上传 PDF / TXT / MD。

第二阶段支持：

- Europe PMC；
- OpenAlex；
- Semantic Scholar；
- Google Scholar Alerts 邮件导入。

### 4.2 中文补充数据源

用于研究中国境内应用情况，建议支持人工检索入口和手动导入：

- CNKI / 中国知网；
- 万方；
- 维普；
- SinoMed 中国生物医学文献服务系统；
- NSTL 国家科技图书文献中心。

中文数据源第一版不做自动爬取，不作为自动主链路。

### 4.3 不作为自动数据源

- genOway 官网不作为数据源；
- Google Scholar 不作为自动抓取源；
- CNKI、万方、维普等中文数据库不作为后台自动爬取源；
- 非正规全文下载渠道不作为数据源。

## 5. 论文状态要求

系统必须维护论文状态，包括：

- 已发现；
- 已获取摘要；
- 摘要缺失；
- 已发现开放全文；
- 已获取全文；
- 全文受限；
- 全文解析成功；
- 全文解析失败；
- 已完成分析；
- 参考价值高；
- 参考价值中；
- 参考价值低；
- 需要人工审核；
- 已人工确认；
- 已排除；
- 手动全文；
- Google Scholar 线索；
- 中文文献线索；
- CAJ 待转换。

## 6. 自动获取摘要需求

系统应支持按关键词、时间范围、DOI、PMID 获取论文题录和摘要。

至少保存：

- title；
- abstract；
- authors；
- journal；
- publication_date；
- year；
- doi；
- pmid；
- pmcid；
- source；
- query_keyword；
- language；
- region_relevance。

## 7. 自动获取全文需求

系统应根据 DOI、PMID、PMCID 或标题尝试寻找合法开放全文。

获取优先级：

1. PMCID 对应的 PMC / Europe PMC XML；
2. Unpaywall 开放 PDF；
3. Semantic Scholar openAccessPdf；
4. OpenAlex open access location；
5. 出版社开放访问页面。

没有合法开放全文时，不下载，只记录状态。

## 8. 手动全文分析需求

用户可以手动下载 PDF 或整理全文后导入系统。

支持格式：

- PDF；
- TXT；
- MD；
- 后续支持 XML / HTML / DOCX。

CAJ 文件可以上传登记，但第一版不直接解析。系统应提示用户使用 CAJViewer 等本地工具转换为 PDF 后再上传。

## 9. 分析需求

系统应从摘要和全文中抽取：

- 是否提到 genOway / 基锘威；
- 模型名称；
- 模型类型；
- 靶点；
- 疾病领域；
- 治疗领域；
- 应用场景；
- 使用机构；
- 机构国家或地区；
- 是否属于中国境内应用线索；
- 证据句；
- 证据所在章节；
- 参考价值；
- 是否需要人工审核。

## 10. 报告需求

系统应输出：

1. 新增论文清单；
2. 高参考价值论文清单；
3. 模型应用分析；
4. 疾病领域分析；
5. 靶点 / 通路分析；
6. 客户机构 / 作者单位分析；
7. 中国境内应用线索；
8. 待人工审核论文清单；
9. 无法获取全文论文清单；
10. 手动导入论文分析结果；
11. 中文文献线索分析；
12. CAJ 待转换清单。

报告格式：

- Excel；
- Markdown；
- CSV。
